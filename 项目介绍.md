# 中文情感分析系统 — 项目介绍

## 1. 项目概述

基于深度学习的**中文文本情感分析系统**。输入任意中文文本，自动输出情感判断（正面/中性/负面），并展示**为什么**做出这个判断。

**核心亮点**：从数据探索→传统基线→预训练微调→可解释性→谐音/讽刺识别→模型集成，完整走通 NLP 项目全流程。特别针对中文的**谐音梗、网络用语、抽象话、反讽表达**做了深度处理，覆盖 2024-2026 年主流互联网语境。

> 📂 GitHub: [https://github.com/fengtangkoukoudi/qjh-nlp](https://github.com/fengtangkoukoudi/qjh-nlp)

## 2. 问题定义

### 2.1 解决的场景

| 输入 | 输出 | 难点类型 |
|------|------|----------|
| "这家店服务好，推荐！" | 😊 正面 | 普通表达 |
| "还行吧，无功无过" | 😐 中性 | 模糊表达 |
| "太差了，态度还恶劣" | 😞 负面 | 普通表达 |
| **"服务真是太好了，等了两个小时"** | 😞 负面 | **讽刺反话** |
| **"集美们避雷！绝绝子的差！"** | 😞 负面 | **谐音+网络语** |
| **"典，这就是你们说的优质服务"** | 😞 负面 | **抽象话+讽刺** |
| **"u1s1，质量yyds，但价格属实刺客"** | 😊 正面 | **拼音缩写+黑话** |
| **"city不city不知道，服务态度窒息"** | 😞 负面 | **中英混搭+网络热词** |

### 2.2 技术挑战

| 挑战 | 说明 | 例子 |
|------|------|------|
| 中文分词 | 无空格分隔 | "服务态度非常好" → 服务态度/非常/好 |
| 谐音/网络用语 | 拼音谐音、缩写、中英混搭 | 针不戳→真不错，yyds→永远的神 |
| 抽象话 | 贴吧/社交平台单字表达 | 典·孝·急·乐·赢·绷 |
| 讽刺/反语 | 字面正面实际负面 | "谢谢你们把我手机摔烂了" |
| 可解释性 | 不只是给结果，要解释为什么 | 哪个词推高了正面/负面 |

## 3. 系统架构

### 3.1 五层统一分析管道

```
用户输入文本
    │
    ▼
┌──────────────────────────────────────────────┐
│  Layer 1: 谐音/网络用语标准化 (200+ 词条, 9大类) │
│  拼音谐音 · 缩写 · 英文谐音梗 · 数字谐音          │
│  贴吧抽象话 · 平台黑话 · 经济消费 · 情绪表达      │
├──────────────────────────────────────────────┤
│  Layer 2: 深度学习情感预测                      │
│  RoBERTa-wwm-ext (94.17%) + BERT + LinearSVM │
│  三分类: 正面 / 中性 / 负面                    │
│  情感强度: 0-100 连续分数                      │
├──────────────────────────────────────────────┤
│  Layer 3: 讽刺检测 (16 种模式, 4 个维度)        │
│  单句模式 · 双句对照 · emoji信号 · 语境评分      │
├──────────────────────────────────────────────┤
│  Layer 4: LIME 可解释性 (jieba 定制分词)        │
│  每个词的贡献度 → 绿色/红色高亮                  │
├──────────────────────────────────────────────┤
│  Layer 5: 多模型集成投票                        │
│  RoBERTa + BERT + LinearSVM 共同决策           │
└──────────────────────────────────────────────┘
    │
    ▼
最终输出: 情感标签 + 强度分数 + 推理过程 + 词级解释 + 多模型对比
```

### 3.2 模型对比

| 模型 | 类型 | 准确率 | 速度 | 参数量 |
|------|------|--------|------|--------|
| **RoBERTa-wwm-ext** | 预训练微调 | **94.17%** | 109ms | 102M |
| BERT-base | 预训练微调 | 91.92% | 109ms | 102M |
| LinearSVM | 传统 ML | 91.83% | <1ms | — |
| **🤝 集成投票** | 三模型融合 | **~94.5%** | 220ms | — |

### 3.3 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 预训练模型 | `hfl/chinese-roberta-wwm-ext` | 全词遮罩，中文 NLU 榜单领先 |
| 分词 | jieba | 中文分词标准工具，同时驱动 LIME |
| 可解释性 | LIME + jieba `split_expression` | 模型无关，定制分词解决中文适配 |
| 谐音标准化 | 自建词典 (200+ 词条, 9 大类) | 覆盖 2024-2026 主流互联网表达 |
| 讽刺检测 | 规则引擎 (16 种模式, 4 维度) | 语言学特征 + 情感矛盾 + emoji 信号 |
| Web 框架 | Streamlit | 纯 Python，快速构建 |
| 深度学习 | PyTorch + HuggingFace Transformers | 生态最完善 |
| 数据集 | ChnSentiCorp | 12,000 条中文评论 |

## 4. 功能特性

系统为统一分析页 + 可选批量模式。**一次输入，全部输出**：

| 功能 | 说明 |
|------|------|
| 🎯 情感结果 | 三分类标签 + 0-100 情感强度 + 确信度等级 |
| 🔤 谐音识别 | 自动检测 200+ 种网络用语并标注标准化结果 |
| 🎭 讽刺翻转 | 16 种模式多维度检测，命中则自动翻转情感 |
| 🤝 多模型对比 | 三个模型各自的判断和置信度 |
| 🔑 LIME 词贡献 | 绿色=推高正面，红色=推高负面，条形图量化 |
| 📋 分析历史 | 自动保存最近 20 条，支持回溯 |
| 📊 批量模式 | 侧边栏一键切换，CSV 导出 + 情感趋势折线图 |
| 🌙 深色模式 | 侧边栏开关 |

## 5. 谐音/讽刺引擎（深化版）

### 5.1 谐音词典：200+ 词条，9 大类别

| 类别 | 数量 | 典型词条 |
|------|------|----------|
| 拼音谐音 | 25 | 针不戳、蓝瘦香菇、集美、酱紫、灰常 |
| 贴吧/抽象话 | 15 | 典、孝、急、乐、赢、绷、难绷、哈人、逆天、抽象 |
| 2024-2026 热词 | 20 | 遥遥领先、显眼包、搭子、city不city、纯纯、i人/e人、尊嘟假嘟 |
| 拼音缩写 | 18 | yyds、xswl、u1s1、srds、xdm、jms、pljj、tql |
| 情绪表达 | 16 | 破防、下头、上头、emo、裂开、社死、心态炸了、难顶 |
| 生活态度 | 18 | 躺平、摆烂、内卷、润、割韭菜、大冤种、塌房、种草 |
| 英文谐音梗 | 14 | book思议、深藏blue、无可phone告、半tour废、duck不必 |
| 平台黑话 | 18 | 键盘侠、杠精、柠檬精、舔狗、海王、社畜、画饼、甩锅 |
| 经济消费 | 10 | 平替、智商税、刺客、雪糕刺客、消费降级 |
| 数字谐音 | 12 | 666、233、555、520、886、7456、748 |

### 5.2 讽刺检测：16 种模式，4 个维度

**A. 单句模式（12 种）**

| 模式 | 示例 |
|------|------|
| 夸张赞美 + 负面语境 | "服务**真是太好了**，等了两个小时" |
| 感谢 + 坏结果 | "**多亏了**快递，一周后**才**到" |
| 谢谢 + 损坏 | "**谢谢**你们，把我手机**摔烂**了" |
| 宣传 vs 实际 | "**号称**旗舰店，**结果**全是假货" |
| 反问式讽刺 | "这**也叫**优质服务？" |
| 质疑式反问 | "**这就是你们说的**服务？" |
| 避雷/劝退 | "**避雷**！千万别买" |
| 反语"有意思" | "**太有意思了**（实际很无语）" |
| 夸张等待 + 坏结果 | "**等了三个小时才**上菜" |
| 夸大 + 但是 | "**完美**的包装，**但是**里面碎了" |
| 夸张负面 | "差得**离谱**"、"慢得**要死**" |
| 贴吧五字真言 | "**典**" "**孝**" "**急**" "**乐**" "**赢**" |

**B. 双句对照（2 种）**

| 模式 | 示例 |
|------|------|
| 时间线反差 | "本来以为很快，结果等了一周" |
| 口碑 vs 实际 | "网上都说好，实际到手发现…" |

**C. Emoji 信号（10 个讽刺 emoji）**
🙃 😅 🤡 💀 🤣 😬 🥲 🤪 😏 🫠

**D. 语境评分**
- 负面信号词 + 高正面置信度 → 矛盾扣分
- 极端正面词 + 负面词共现 → 疑似反讽
- 连续问号 "？？" → 质疑语气
- 省略号 "…" + 负面词 → 意味深长

## 6. 实现过程

| Phase | 文件 | 内容 |
|-------|------|------|
| 1. 数据探索 | `01_explore_data.py` | 标签分布、文本长度分析、可视化 |
| 2. ML 基线 | `02_baseline_ml.py` | TF-IDF + LinearSVM → 91.83% (~5s) |
| 3. RoBERTa 微调 | `03_train_bert.py` | roberta-wwm-ext → 94.17% (~4min GPU) |
| 4. 评估对比 | `04_evaluate.py` | 三模型对比、混淆矩阵、错误分析 |
| 5. 可解释性 | `06_explain.py` | LIME 词贡献 + TextRank 关键词 |
| 6. 谐音/讽刺 | `07_slang_normalizer.py` | 200+ 词条、16 种讽刺模式 |
| 7. Web 界面 | `app.py` | 统一分析管道、深色模式、批量切换 |

训练配置：`max_length=256, batch_size=16, lr=3e-5, epochs=8`，梯度累积 + 早停(patience=3) → 3.67 epochs 收敛。

## 7. 实验结果

### 7.1 核心指标

| 指标 | LinearSVM | BERT-base | RoBERTa-wwm |
|------|-----------|-----------|-------------|
| 准确率 | 91.83% | 91.92% | **94.17%** |
| Macro F1 | 91.83% | 91.91% | **94.16%** |
| 推理速度 | <1ms | 109ms | 109ms |

### 7.2 深度理解测试

| 类型 | 输入 | 检测 | 结果 |
|------|------|------|------|
| 谐音 | "针不戳，绝绝子" | 🔤 谐音×2 | 😊 正面 ✅ |
| 谐音+负面 | "蓝瘦香菇，等了一周" | 🔤 谐音×2 | 😞 负面 ✅ |
| 抽象话 | "典，这就是优质服务" | 🔤🎭 翻转 | 😞 负面 ✅ |
| 讽刺-等待 | "服务太好了，等了两个小时" | 🎭 翻转 | 😞 负面 ✅ |
| 讽刺-反话 | "多亏了快递，一周后才到" | 🎭 翻转 | 😞 负面 ✅ |
| 讽刺-感谢 | "谢谢你们，手机摔烂了" | 🎭 翻转 | 😞 负面 ✅ |
| 讽刺-名不副实 | "号称旗舰店，结果假货" | 🎭 翻转 | 😞 负面 ✅ |
| 讽刺-反问 | "这就是优质服务？有意思" | 🎭 翻转 | 😞 负面 ✅ |
| 混合 | "集美们避雷！绝绝子的差" | 🔤🎭 翻转 | 😞 负面 ✅ |
| 混合 | "遥遥领先的配送，三天没出仓" | 🔤 识别 | 😞 负面 ✅ |
| 正常 | "味道好价格实惠" | — | 😊 正面 ✅ |

### 7.3 关键发现

1. **RoBERTa-wwm-ext 比 BERT-base 高 2.25%**：全词遮罩对中文提升显著
2. **规则引擎 + 深度学习互补**：纯模型对讽刺的召回率低，规则引擎弥补了短处
3. **LIME 默认分词器对中文完全无效**：必须用 jieba 定制 `split_expression`
4. **抽象话需单独处理**："典/孝/急" 等单字情绪浓烈，但模型会将其当作普通字忽略
5. **集成投票一致性高**：多数 case 三模型 100% 一致，不一致的恰是疑难样本

## 8. 创新点

1. **LIME + jieba 深度整合**：用 jieba 定制 `split_expression`，解决默认空格分词对中文完全失效的问题
2. **200+ 谐音/网络用语标准化管道**：9 大类别覆盖拼音谐音、缩写、英文谐音梗、抽象话、平台黑话，预处理后再送入模型
3. **16 种讽刺模式 × 4 维度检测**：单句 + 双句对照 + emoji 信号 + 语境评分，多引擎综合判断
4. **模型无关的讽刺翻转**：检测到讽刺信号后自动翻转情感极性，不依赖标注数据
5. **三模型集成投票**：RoBERTa + BERT + LinearSVM，DL 与 ML 互补
6. **二分类→三分类阈值法**：无需三分类标注，用置信度阈值扩展出中性类别
7. **情感强度量化**：0-100 连续分数 + 5 级确信度
8. **全流程闭环**：数据探索→基线→微调→评估→标准化→讽刺检测→可解释性→集成→部署

## 11. 改进方向

1. **讽刺分类器升级**：规则引擎 → 用 ChatGLM 标注讽刺样本 → 微调专用讽刺分类器
2. **方面级情感分析**：抽取"服务""性价比""物流"等具体方面分别判断情感

6. **大模型蒸馏**：用 Qwen/ChatGLM 标注困难样本 → 蒸馏到小模型
